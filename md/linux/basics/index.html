    Linux Temel Bilgiler¶

    Temel Sistem Kavramları¶

    Kavram Açıklama
    # Terminalde root kullanıcısını simgeler. Shell betiklerinde yorum satırıdır.
    $ Terminalde sınırlı yetkiye sahip standart kullanıcıları simgeler.
    Soft Link (Sembolik Link) Hedef dosyanın yolunu referans alan yönlendiricidir; asıl dosya silinirse işlevini yitirir.
    Hard Link (Sabit Link) Dosyanın diskteki veri bloklarına doğrudan bağlanan kopyasıdır; asıl dosya silinse de veriye erişim devam eder.
    inode (İndis Düğümü) Dosya sisteminde veri bloklarının konumunu ve dosya özniteliklerini saklayan benzersiz kimlik numarasıdır.
    Daemon (Arka Plan Süreci) Sistem açılışında başlayan ve kullanıcı etkileşimi olmadan arka planda çalışan servislerdir.
    Process (Süreç) Bellekte yürütülmekte olan, belirli bir yaşam döngüsüne sahip aktif program veya komut oturumudur.
    CPU Central Processing Unit
    RAM Random Access Memory
    RootFS Root Filesystem
    Scheduler (Zamanlayıcı) CPU kaynaklarını belirli algoritmalar çerçevesinde process'ler arasında paylaştıran kernel alt sistemidir.
    Polling (Sorgulama) İşlemcinin bir donanım veya I/O aygıtının durumunu belirli aralıklarla aktif olarak kontrol etmesi yöntemidir.

    Dikkat

    Linux büyük/küçük harf duyarlıdır. File.txt ile file.txt farklı dosyalardır.

    G/Ç Yönlendirme Operatörleri¶

    Operatör Adı Açıklama
    > Stdout Yönlendirme Çıktıyı dosyaya yazar; dosya varsa üzerine yazar.
    >> Stdout Ekleme Çıktıyı dosyanın sonuna ekler; mevcut veriyi korur.
    < Stdin Yönlendirme Stdin'i klavye yerine dosyadan alır.
    2> Stderr Yönlendirme Yalnızca hata mesajlarını (stderr) dosyaya yönlendirir.
    &> Birleşik Yönlendirme Hem stdout hem stderr'ı aynı dosyaya yönlendirir.
    tee Çift Yönlü Çıktı Çıktıyı hem terminale basar hem dosyaya yazar. -a ile ekler.
    ; Ardışık Çalıştırma Komutları sırayla çalıştırır; başarı durumu gözetilmez.
    & Arka Plan İşlemi Komutu arka planda çalıştırır; terminal serbest kalır.
    \| Pipeline Bir komutun çıktısını bir sonraki komutun girdisine bağlar. 
    1

    sleep 10 &                            # Arka planda çalıştır
echo "merhaba" > dosya.txt           # Dosyaya yaz (üzerine yazar)
ls >> dosya.txt                       # Dosyaya ekle
cat < dosya.txt                       # Dosyadan oku
telnet localhost 2> hata.txt          # Hataları dosyaya yönlendir
echo "merhaba" | tee -a dosya.txt    # Hem ekrana hem dosyaya yaz
komut1 ; komut2 ; komut3              # Sırayla çalıştır

    Regular Expression (Regex)¶

    BRE (Basic Regular Expression): grep, sed gibi araçların varsayılan modudur. +, ?, |, (), {} gibi özel karakterlerin önüne \ koymak gerekir.

    ERE (Extended Regular Expression): grep -E, egrep, awk araçlarında kullanılır. Özel karakterler \ gerektirmeden doğrudan kullanılabilir.

    Temel Karakterler¶

    Karakter Anlamı Örnek Eşleşen
    . Herhangi bir karakter a.c abc, axc, a1c
    * Önceki karakterden 0 veya daha fazla ab*c ac, abc, abbc
    + Önceki karakterden 1 veya daha fazla (ERE) ab+c abc, abbc
    ? Önceki karakter opsiyonel (ERE) colou?r color, colour
    ^ Satır başı ^Hata "Hata" ile başlayan satırlar
    $ Satır sonu Hata$ "Hata" ile biten satırlar
    [] Karakter sınıfı [abc] a, b veya c
    [^] Hariç tutma [^0-9] Rakam olmayan her karakter
    {n,m} n ile m arası tekrar a{2,4} aa, aaa, aaaa
    () Gruplama (ab)+ ab, abab, ababab
    \| Alternatif (veya) kedi\|köpek kedi veya köpek
    \d (GNU) Rakam \d{3} 123
    \w Kelime karakteri (harf, rakam, _) \w+ merhaba_123
    \s Boşluk karakteri \s+ boşluk, tab

    POSIX Karakter Sınıfları¶

    Sınıf Anlamı
    [[:digit:]] Rakamlar (0–9)
    [[:alpha:]] Harfler
    [[:alnum:]] Harf ve rakamlar
    [[:space:]] Boşluk karakterleri
    [[:upper:]] Büyük harfler
    [[:lower:]] Küçük harfler
    [[:punct:]] Noktalama işaretleri

    Dizin ve Dosya Yapısı¶

    Dosya Türleri¶

    ls -l çıktısında en baştaki karakter dosya türünü belirtir:

    Karakter Tür
    - Düzenli dosya (Regular File)
    d Dizin (Directory)
    l Sembolik Link (Symbolic Link)
    c Karakter Aygıtı (Character Device — terminal, seri port)
    b Blok Aygıtı (Block Device — disk, USB)
    s Soket (Socket)
    p Adlandırılmış Boru Hattı (Named Pipe / FIFO)

    Gizli Dosyalar

    Adı . (nokta) ile başlayan dosya ve dizinler gizlidir; ls -a ile görüntülenebilir.

    Kök Dizin Yapısı¶

    Dizin Açıklama
    / Tüm dosya sisteminin kökü.
    /bin, /sbin Sistem açılışı için kritik komutlar. Modern dağıtımlarda /usr/bin'e symlink.
    /etc Tüm statik sistem yapılandırma dosyaları (ağ, kullanıcı, güvenlik).
    /usr Paylaşılan kullanıcı programları ve kütüphaneler (salt okunur).
    /lib, /lib64 Dinamik kütüphaneler ve çekirdek modülleri (/lib/modules/<versiyon>/).
    /dev Donanım aygıt düğümleri — UART, I2C, disk vb. kernel'in userspace arayüzü.
    /proc Çekirdek runtime durumunun sanal görünümü (procfs).
    /sys Çekirdek nesne modelinin userspace arayüzü (sysfs).
    /var Çalışma zamanında değişen kalıcı veriler — loglar (/var/log), state (/var/lib).
    /tmp Geçici dosyalar; genellikle RAM'de (tmpfs). Yeniden başlatmada silinir.
    /boot Kernel image, DTB, initramfs gibi önyükleme dosyaları.
    /opt Üçüncü taraf yazılım paketleri.
    /home Kullanıcı ev dizinleri.
    /mnt, /media Harici medya ve geçici bağlama noktaları.
    /run Geçici çalışma süresi bilgileri (PID dosyaları, kilitler).

    /proc — Debug İçin Faydalı Dosyalar

    Dosya İçerik
    /proc/cmdline Kernel başlatma parametreleri
    /proc/meminfo Bellek kullanım bilgisi
    /proc/cpuinfo İşlemci bilgisi
    /proc/<pid>/ Belirli bir process'in detayları

    Sistem Günlükleri¶

    Linux'ta sistem olayları /var/log dizininde saklanır. systemd kullanan sistemlerde journalctl ile de günlüklere erişilebilir.

    Kaynak Açıklama
    /var/log/boot.log Önyükleme sırasındaki mesajlar.
    /var/log/auth.log Kimlik doğrulama ve güvenlik olayları (giriş denemeleri, sudo).
    /var/log/syslog Genel sistem mesajları (Debian/Ubuntu).
    /var/log/messages Genel sistem mesajları (RHEL/CentOS).
    /var/log/kern.log Kernel'e ait detaylı kayıtlar.
    dmesg Kernel ring buffer çıktısı.
    journalctl systemd journal kayıtlarını okur. 
    1
2
3
4
    journalctl -b              # Son önyüklemeden itibaren tüm loglar
journalctl -u <servis>     # Belirli bir servisin logları
journalctl -f              # Canlı log takibi (tail -f benzeri)
dmesg | grep -i error      # Kernel hata mesajlarını filtrele

    Run Levels¶

    SysV init sisteminde çalışma seviyeleri, hangi servislerin aktif olacağını belirler.

    Run Level Anlamı
    0 Kapatma (shutdown)
    1 Tek kullanıcı modu — sistem bakım/onarım
    2 Çoklu kullanıcı, ağ kapalı (dağıtıma göre değişir)
    3 Çoklu kullanıcı, ağ aktif
    4 Tanımlanmamış / Kullanılmaz
    5 Çoklu kullanıcı + grafik arayüz (X11/Wayland) + ağ
    6 Yeniden başlatma (reboot) 
    1
    sudo init 3    # Run level değiştir (örnek: seviye 3)

    Systemd Karşılıkları

    Modern dağıtımlarda SysV run level yerine systemd target kullanılır:

    Run Level Systemd Target
    1 rescue.target
    3 multi-user.target
    5 graphical.target
    0 poweroff.target
    6 reboot.target 
    1
    systemctl isolate multi-user.target

    Device Tree¶

    Gömülü Linux sistemlerinde (ARM tabanlı kartlar gibi) donanımı çekirdeğe tanıtmak için kullanılır.

    Format Açıklama
    .dts Device Tree Source — insan tarafından okunabilir metin formatı.
    .dtb Device Tree Blob — derlenmiş ikili format.

    Çalışma prensibi:

    1. Donanım özellikleri .dts dosyasında tanımlanır.
    2. dtc derleyicisi ile .dtb dosyasına derlenir.
    3. Kernel açılışta bu veriyi okur, uygun sürücüyü yükler ve donanımı yapılandırır.
    1
2
    dtc -I dts -O dtb -o output.dtb input.dts    # .dts → .dtb derleme
dtc -I dtb -O dts -o output.dts input.dtb    # .dtb → .dts tersine çevirme

    Driver (Sürücü)¶

    Linux çekirdeği ile donanım arasında köprü kuran yazılım bileşenidir. Yeni bir cihaz eklendiğinde ilgili sürücü kernel modülü (.ko dosyası) olarak yüklenir.

    Komut Açıklama
    lsmod Yüklü kernel modüllerini listeler.
    modprobe <modül> Kernel modülü yükler (bağımlılıkları da yükler).
    rmmod <modül> Kernel modülünü kaldırır.
    modinfo <modül> Modül hakkında bilgi gösterir.

    Terminal Kısayolları¶

    Kısayol İşlev
    Ctrl + C Çalışan komutu sonlandırır.
    Ctrl + Z Çalışan komutu arka plana alır (duraklatır).
    Ctrl + R Komut geçmişinde arama yapar.
    Ctrl + U İmlecin solundaki tüm karakterleri siler.
    Ctrl + S Terminal çıktı akışını durdurur.
    Ctrl + Q Durdurulan terminal akışını sürdürür.
    Alt + F2 Komut çalıştırma penceresini açar (grafik ortam).
